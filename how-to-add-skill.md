# How to Add a New Skill

Гайд по добавлению нового скилла в этот репозиторий.

---

## 0. Прежде чем добавлять

Спроси себя:

- **Новая ли это тема, или расширение существующей?**
  Если у тебя уже есть `banking-contracts/murabaha.md`, и ты хочешь разобрать «мурабаху для финансирования автомобиля», — это, скорее, **подраздел существующего скилла** или новый скилл в `personal-finance/`, а не дубль.

- **Есть ли источники?**
  Минимум один источник из `sources/registry.md` нужен для любой существенной темы. Если по теме нет ни одного авторитетного источника — это плохой кандидат на скилл.

- **Подходит ли тема под формат?**
  Скиллы — это знания, а не новости. Если что-то меняется чаще раза в год (например, актуальная процентная маржа в продукте банка) — это не должно быть в самом скилле, а должно быть в подключаемой ссылке или в `kazakhstan/*-current.md` (с явной датой ревизии).

## 1. Выбери папку

| Куда положить | Когда |
|---------------|-------|
| `skills/foundations/` | Принцип фикха муамалят (риба, гарар, майсир, кавáид) |
| `skills/banking-contracts/` | Конкретный исламский контракт (мурабаха, мушарака, иджара…) |
| `skills/sukuk/` | Что-то про сукук |
| `skills/equity-screening/` | Скрининг акций, методология AAOIFI |
| `skills/personal-finance/` | Общая теория личных финансов (ипотека как инструмент, бюджет как концепция) |
| `skills/zakat/` | Закят с конкретного типа имущества |
| `skills/crypto-digital/` | Крипта, цифровые активы |
| `skills/ethical-investing/` | ESG, SRI, сравнение с исламскими подходами |
| `skills/waqf/` | Вакф |
| `skills/personal/` | **Операционный скилл**: workflow для личных расчётов и решений |
| `skills/kazakhstan/` | **Локальный**: про конкретный казахстанский продукт, институт, регулирование |

> **Если сомневаешься** — между общей теорией и казахстанской спецификой делай **два скилла**: один в общей категории, второй в `kazakhstan/`. Это лучше, чем размазывать.

---

## 2. Скопируй шаблон

```bash
cp templates/skill-template.md skills/<категория>/<имя-скилла>.md
```

**Имя файла:** строчные буквы, дефисы вместо пробелов, кириллица **не используется**.

✅ Хорошо: `murabaha.md`, `zakat-on-stocks.md`, `adcb-islamic-bank.md`, `portfolio-review-checklist.md`
❌ Плохо: `Мурабаха.md`, `ZakatStocks.MD`, `портфолио ревью.md`

---

## 3. Заполни frontmatter

Открой скопированный файл, отредактируй YAML-блок в начале:

```yaml
---
title: Мурабаха                       # Человеческое название
arabic: مرابحة                        # Арабское написание (если есть)
transliteration: murābaḥa             # Академическая транслитерация
category: banking-contracts           # Должна совпадать с папкой
status: draft                         # draft → stable → needs-review
last-reviewed: 2026-05-10             # Сегодняшняя дата
madhhab-priority: hanafi              # hanafi | none | open
divergence-note: ""                   # Если есть мазхабные различия
related-skills:
  - banking-contracts/musharaka.md
  - foundations/riba.md
key-sources:
  - AAOIFI-SS-08
  - USMANI-INTRO-CH4
applicable-jurisdiction: global       # global | kazakhstan | other
---
```

### Особо про `madhhab-priority`

- `hanafi` — есть устойчивая ханафитская позиция. Используется по умолчанию для большинства скиллов.
- `none` — позиция консенсусная (иджма) или единая по всем мазхабам.
- `open` — современный вопрос без устоявшейся позиции (например, NFT, некоторые DeFi-протоколы).

---

## 4. Заполни секции

Минимум для **стабильного** скилла:

- ⭐ Краткое определение (1–2 предложения)
- ⭐ Шариатская основа
- ⭐ Экономический механизм
- ⭐ Применимость в Казахстане (для практических скиллов)
- ⭐ Что спросить у шариатского советника (для практических скиллов)
- Дополнительное чтение

Для **черновика** (status: draft) можно начать с краткого определения и шариатской основы, постепенно дополняя.

---

## 5. Добавь источники в `sources/registry.md`

Если ты используешь источник, которого ещё нет в реестре, — добавь его туда **до** ссылки в скилле. Принцип: «если ID источника не существует в реестре, он не существует в репозитории».

См. формат в самом `sources/registry.md`.

---

## 6. Обнови `INDEX.md`

Открой `INDEX.md` и добавь строку в соответствующий раздел:

```markdown
| `skills/banking-contracts/murabaha.md` | "мурабаха", "продажа с наценкой", "товарное финансирование" | stable |
```

**Триггеры** — это слова и фразы, по которым Claude должен понять, что нужен этот скилл. Подумай, как пользователь может задать вопрос, и подбери 3–6 формулировок.

---

## 7. Обнови Project Instructions (если триггеры новые или важные)

Если ты добавил скилл, который меняет основной workflow (например, новый `personal/`-скилл для частых сценариев), нужно зеркально обновить инструкцию в Claude Project. См. `docs/project-instructions.md` — там ведётся актуальная версия инструкции.

Минорные дополнения (новые скиллы в существующих категориях) можно копить и синхронизировать партиями (например, раз в месяц).

---

## 8. Закоммить и запушь

```bash
git add skills/<категория>/<имя>.md INDEX.md sources/registry.md
git commit -m "Add skill: <название>"
git push
```

После пуша Claude сможет фетчить файл по raw-URL.

---

## Чек-лист перед коммитом

- [ ] Имя файла — строчные латинские буквы и дефисы.
- [ ] Frontmatter заполнен полностью.
- [ ] `madhhab-priority` выставлен.
- [ ] `last-reviewed` — сегодняшняя дата.
- [ ] Все источники в `key-sources` существуют в `sources/registry.md`.
- [ ] Шариатская основа имеет конкретные ссылки на источники.
- [ ] Секция «Применимость в Казахстане» заполнена (для практических скиллов).
- [ ] `INDEX.md` обновлён.
- [ ] Проверено форматирование Markdown (списки, таблицы, заголовки).

---

## Антипаттерны: чего не делать

- ❌ **Не добавлять «универсальные» скиллы** уровня «всё об исламских финансах» — это будет дубль с другими скиллами.
- ❌ **Не копировать большие куски текста из источников** — это нарушает копирайт. Перефразируй своими словами и давай ссылку.
- ❌ **Не указывать спорные мнения как «единственно верные»** — даже если они тебе нравятся. Соблюдай иерархию источников.
- ❌ **Не вносить личные данные** (свои цифры, портфель, бюджет) в публичный репозиторий. Этому место в Project Knowledge Claude.
- ❌ **Не оставлять `status: draft` навечно** — либо доводить до `stable`, либо удалять.
